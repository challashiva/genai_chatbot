Backend — input validation

Add this at the top of chat_stream() in api/chat.py before calling ai_service:

# Validate messages is not empty
if not request.messages:
    raise HTTPException(
        status_code=400,
        detail="Messages cannot be empty"
    )

# Validate individual message lengths
for msg in request.messages:
    if len(msg.content) > 2000:
        raise HTTPException(
            status_code=400,
            detail=f"Message too long ({len(msg.content)} chars). Max 2000."
        )

Backend — logging

Add to the top of ai_service.py and inside stream_chat_response():

# At top of file
import logging
import time

logger = logging.getLogger(__name__)

# Inside stream_chat_response(), before the for loop:
total_chars = sum(len(m.content) for m in messages)
logger.info(f"Request: {len(messages)} messages, {total_chars} total chars")
start = time.time()
token_count = 0

# Inside the for loop, after yield text:
token_count += 1

# After the for loop ends:
elapsed = round(time.time() - start, 2)
logger.info(f"Done: {elapsed}s, {token_count} tokens")

Frontend — auto-grow textarea

Replace the onChange on the textarea in ChatWindow.jsx:

onChange={(e) => {
  setInputText(e.target.value)
  // Reset height first so it can shrink
  e.target.style.height = 'auto'
  // Set height to content, max 6 rows (6 * 24px)
  e.target.style.height = Math.min(e.target.scrollHeight, 144) + 'px'
}}

Frontend — copy button for messages

Add this inside MessageBubble.jsx — only shown on assistant messages:

const [copied, setCopied] = useState(false)

const handleCopy = async () => {
  await navigator.clipboard.writeText(content)
  setCopied(true)
  setTimeout(() => setCopied(false), 2000)
}

// In JSX, below the bubble div — only for assistant:
{!isUser && (
  <button onClick={handleCopy} style={{
    background: 'none', border: 'none',
    fontSize: '11px', color: '#9ca3af',
    cursor: 'pointer', marginTop: '4px', marginLeft: '40px'
  }}>
    {copied ? '✓ Copied' : 'Copy'}
  </button>

Frontend — stop streaming with AbortController

The most complex improvement. Changes span api/chat.js, useChat.js, and ChatWindow.jsx:

// --- api/chat.js: add signal parameter ---
export async function streamChat({ messages, onChunk, onDone, onError, signal }) {
  const response = await fetch('/api/chat/stream', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ messages }),
    signal,  // ← add this
  })
  ...
}

// --- useChat.js: add AbortController ---
const abortRef = useRef(null)

const sendMessage = useCallback(async (userText) => {
  abortRef.current = new AbortController()
  await streamChat({ ..., signal: abortRef.current.signal })
}, [...])

const stopStreaming = useCallback(() => {
  abortRef.current?.abort()
  finishStreaming()
}, [finishStreaming])

// --- ChatWindow.jsx: show Stop button ---
{isLoading && (
  <button onClick={stopStreaming}>Stop</button>
)}
